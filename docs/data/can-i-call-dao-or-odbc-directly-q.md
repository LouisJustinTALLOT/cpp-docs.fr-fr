# Puis-je appeler DAO ou ODBC directement&#160;?
[!INCLUDE[vs2017banner](../assembler/inline/includes/vs2017banner.md)]

Comme c'est généralement le cas en environnement MFC, si vous avez besoin d'un contrôle plus fin, vous pouvez appeler DAO ou ODBC directement mais aussi y accéder par les classes.  MFC tente de simplifier la programmation pour Windows, mais reste aussi à l'écart si vous avez besoin d'accéder aux interfaces API sous\-jacentes.
