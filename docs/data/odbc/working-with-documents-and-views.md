# <a name="working-with-documents-and-views"></a>Utilisation des documents et vues

La bibliothèque Microsoft Foundation Classes (MFC) s’appuie sur une architecture document/vue pour la plupart de ses fonctionnalités. En règle générale, un document stocke vos données et une vue les affiche dans la zone cliente d’une fenêtre frame et gère l’interaction de l’utilisateur avec les données. La vue communique avec le document pour obtenir et mettre à jour les données. Vous pouvez utiliser les classes de base de données avec le Framework ou sans celui-ci.

Pour plus d’informations sur l’utilisation des classes de base de données dans l’infrastructure, consultez [MFC : utilisation de classes de base de données avec des documents et des vues](../../data/mfc-using-database-classes-with-documents-and-views.md).

Par défaut, l’Assistant Application MFC crée une application squelette sans prise en charge de base de données. Toutefois, vous pouvez sélectionner des options pour inclure la prise en charge minimale des bases de données ou une prise en charge plus complète des formulaires. Pour plus d’informations sur les options de l’Assistant Application, consultez [prise en charge des bases de données, Assistant Application MFC](../../mfc/reference/database-support-mfc-application-wizard.md).

Vous pouvez également utiliser les classes de base de données sans utiliser l’architecture document/vue complète. Pour plus d’informations, consultez [MFC : utilisation de classes de bases de données sans document ni vue](../../data/mfc-using-database-classes-without-documents-and-views.md).
