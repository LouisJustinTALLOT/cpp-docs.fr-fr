# Modifying the Layout Grid
[!INCLUDE[vs2017banner](../assembler/inline/includes/vs2017banner.md)]

Lorsque vous placez ou organisez des contrôles dans une boîte de dialogue, vous pouvez utiliser la grille pour un positionnement plus précis.  Lorsque la grille est activée, les contrôles apparaissent comme « alignés » sur les lignes en pointillés de la grille comme si elle était aimantée.  Vous pouvez activer ou désactiver cette fonctionnalité « aligner sur la grille » et modifier la taille des cellules de la grille.  
  
### Pour activer ou désactiver la grille  
  
1.  Dans le menu **Format**, cliquez sur **Paramètres du repère**.  
  
2.  Dans la [boîte de dialogue Paramètres du repère](../mfc/guide-settings-dialog-box.md), activez ou désactivez le bouton **Grille**.  
  
     Vous pouvez toujours contrôler la grille dans les fenêtres individuelles de l'Éditeur de boîtes de dialogue en utilisant le bouton **Activer\/Désactiver la grille** dans la [barre d'outils Éditeur de boîtes de dialogue](../mfc/showing-or-hiding-the-dialog-editor-toolbar.md).  
  
### Pour changer la taille de la grille  
  
1.  Dans le menu **Format**, cliquez sur **Paramètres du repère**.  
  
2.  Dans la [boîte de dialogue Paramètres du repère](../mfc/guide-settings-dialog-box.md), tapez la hauteur et la largeur en unités de boîte de dialogue pour les cellules de la grille.  La hauteur ou la largeur minimale est 4 unités de boîte de dialogue.  Pour plus d'informations sur les unités de boîte de dialogue, consultez [Organisation des contrôles dans les boîtes de dialogue](../mfc/arrangement-of-controls-on-dialog-boxes.md).  
  
 Pour plus d'informations sur l'ajout de ressources aux projets managés, consultez [Ressources dans les applications](../Topic/Resources%20in%20Desktop%20Apps.md) dans le *Guide du développeur .NET Framework.* Pour obtenir des informations sur l'ajout de fichiers de ressources aux projets managés, l'accès aux ressources, l'affichage de ressources statiques et l'assignation de chaînes de ressources aux propriétés, et ce manuellement, consultez [Procédure pas à pas : localisation des applications Windows Forms](http://msdn.microsoft.com/fr-fr/9a96220d-a19b-4de0-9f48-01e5d82679e5) et [Walkthrough: Using Resources for Localization with ASP.NET](../Topic/Walkthrough:%20Using%20Resources%20for%20Localization%20with%20ASP.NET.md).
