# Editing Properties for an ActiveX Control
[!INCLUDE[vs2017banner](../assembler/inline/includes/vs2017banner.md)]

Les contrôles ActiveX fournis par des fabricants indépendants peuvent intégrer leurs propres propriétés et caractéristiques.  Les propriétés des contrôles ActiveX sont affichées dans la fenêtre Propriétés.  De plus, les pages de propriétés créées par les auteurs du contrôle ActiveX sont affichées dans la boîte de dialogue **Pages de propriétés** \(pour afficher la page de propriétés pour un contrôle ActiveX spécifique, cliquez sur le bouton Page de propriétés dans la [fenêtre Propriétés](../Topic/Properties%20Window.md)\).  
  
 Plusieurs onglets s'affichent dans la page de propriétés pour un contrôle ActiveX, en fonction des pages de propriétés qui font partie du contrôle ActiveX.  
  
> [!NOTE]
>  La procédure suivante s'applique à l'utilisation de la page de propriétés pour modifier des contrôles ActiveX.  Vous pouvez également rechercher et modifier les propriétés ActiveX dans la nouvelle fenêtre Propriétés.  
  
### Pour modifier les propriétés d'un contrôle ActiveX  
  
1.  Sélectionnez le contrôle **ActiveX**.  
  
2.  Dans le menu **Affichage**, cliquez sur **Page de propriétés** et affichez les propriétés.  
  
3.  Apportez les modifications souhaitées à la page de propriétés.  
  
 Pour plus d'informations sur l'ajout de ressources aux projets managés, consultez [Ressources dans les applications](../Topic/Resources%20in%20Desktop%20Apps.md) dans le *Guide du développeur .NET Framework.* Pour obtenir des informations sur l'ajout de fichiers de ressources aux projets managés, l'accès aux ressources, l'affichage de ressources statiques et l'assignation de chaînes de ressources aux propriétés, et ce manuellement, consultez [Procédure pas à pas : localisation des applications Windows Forms](http://msdn.microsoft.com/fr-fr/9a96220d-a19b-4de0-9f48-01e5d82679e5) et [Walkthrough: Using Resources for Localization with ASP.NET](../Topic/Walkthrough:%20Using%20Resources%20for%20Localization%20with%20ASP.NET.md).  
  
 Configuration requise  
  
 Win32
