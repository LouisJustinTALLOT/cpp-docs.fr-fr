# Affichage ou masquage de la barre d’outils &#201;diteur de bo&#238;tes de dialogue
[!INCLUDE[vs2017banner](../assembler/inline/includes/vs2017banner.md)]

Lorsque vous ouvrez l'Éditeur de boîtes de dialogue, la barre d'outils Éditeur de boîtes de dialogue s'affiche automatiquement en haut de votre solution.

 La barre d'outils Éditeur de boîtes de dialogue contient des boutons pour organiser la disposition des contrôles dans la boîte de dialogue, par exemple la taille et l'alignement.  Les boutons de la barre d'outils Éditeur de boîtes de dialogue correspondent aux commandes du menu Format.  Pour plus d'informations, consultez [Touches accélérateur pour l'Éditeur de boîtes de dialogue](../mfc/accelerator-keys-for-the-dialog-editor.md).  
  
 Lorsque vous vous trouvez dans l'Éditeur de boîtes de dialogue, vous pouvez activer et désactiver l'affichage de la barre d'outils Éditeur de boîtes de dialogue en la sélectionnant dans la liste des barres d'outils et des fenêtres disponibles.  
  
### Pour afficher ou masquer la barre d'outils Éditeur de boîtes de dialogue  
  
1.  Dans le menu **Affichage**, cliquez sur **Barres d'outils**, puis choisissez **Éditeur de boîtes de dialogue** dans le sous\-menu.  
  
    > [!NOTE]
    >  La barre d'outils Éditeur de boîtes de dialogue s'affiche par défaut lorsque vous ouvrez une ressource de boîte de dialogue dans l'Éditeur de boîtes de dialogue ; cependant, si vous fermez cette barre d'outils, vous devrez l'appeler la prochaine fois que vous ouvrez une ressource de boîte de dialogue.  
  
 Pour plus d'informations sur l'ajout de ressources aux projets managés, consultez [Ressources dans les applications](../Topic/Resources%20in%20Desktop%20Apps.md) dans le *Guide du développeur .NET Framework.* Pour obtenir des informations sur l'ajout de fichiers de ressources aux projets managés, l'accès aux ressources, l'affichage de ressources statiques et l'assignation de chaînes de ressources aux propriétés, et ce manuellement, consultez [Procédure pas à pas : localisation des applications Windows Forms](http://msdn.microsoft.com/fr-fr/9a96220d-a19b-4de0-9f48-01e5d82679e5) et [Walkthrough: Using Resources for Localization with ASP.NET](../Topic/Walkthrough:%20Using%20Resources%20for%20Localization%20with%20ASP.NET.md).  
  
 Configuration requise  
  
 Win32
