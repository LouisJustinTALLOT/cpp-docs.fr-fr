# Aucun &#39;&lt;nom_proc&#233;dure&gt;&#39; accessible non g&#233;n&#233;rique trouv&#233;.
Une instruction appelle une procédure qui possède plusieurs versions surchargées, mais toutes les versions surchargées sont génériques et l’appel ne fournit pas d’arguments de type.  
  
 S’il n’existe qu’une seule version générique et si vous l’appelez sans argument de type, le compilateur peut tenter une *inférence de type*. Pour plus d’informations, consultez « Inférence de type » dans [Generic Procedures in Visual Basic](../Topic/Generic%20Procedures%20in%20Visual%20Basic.md). Toutefois, s’il existe plusieurs versions génériques, le compilateur n’est pas en mesure de choisir entre eux, sauf si vous fournissez des arguments de type. Si vous fournissez un argument de type, vous devrez le faire pour chaque paramètre de type défini par l’une des versions surchargées.  
  
 **ID d’erreur :** BC32117  
  
### Pour corriger cette erreur  
  
-   Appelez la procédure avec une liste d’arguments de type.
