# CA0060
Impossible de charger les assemblys référencés indirectement.  
  
 Cet avertissement se produit lorsque l'outil d'analyse ne parvient pas à charger les assemblys référencés indirectement par l'assembly en cours d'analyse.  Une « référence indirecte » désigne un assembly de référence auquel l'une des références de votre assembly d'analyse fait référence.  Par exemple si l'analyse du code \(FxCop\) analyse l'assembly A, qui fait référence à l'assembly B, qui à son tour fait référence à l'assembly C mais l'assembly A ne fait pas référence à l'assembly C, alors ce dernier est une référence indirecte et l'assembly B est une référence directe.  
  
 Cela peut entraîner l'erreur CA0001 à d'autres endroits car de nombreux états inattendus peuvent se produire dans ce cas.  Si vous utilisez l'outil d'analyse du code managé à partir de [!INCLUDE[vsprvs](../assembler/masm/includes/vsprvs_md.md)], voici quelques motifs susceptibles d'entraîner un avertissement CA0060 :  
  
-   L'assembly d'entrée a été remplacé et pointe désormais vers un assembly qui fait référence à des assemblys autres que ceux indiqués dans la liste de références du fichier de projet [!INCLUDE[vsprvs](../assembler/masm/includes/vsprvs_md.md)].  
  
-   Si un projet est modifié pour appeler une étape de build personnalisée avant que FxCopCmd.exe s'exécute et que le processus de génération personnalisée ajoute d'autres références à l'assembly d'entrée, l'avertissement CA0060 est généré.  
  
-   En C\+\+, vous pouvez référencer des assemblys à l'aide de ForcedUsing dans VCPROJ, en remplaçant la ligne de commande pour transmettre \/FU ou en ajoutant \#using \<Some.dll\> dans le code source uniquement.  
  
-   Un assembly tiers qui a des références privées à d'autres assemblys dont vous n'avez pas besoin pour compiler et exécuter votre code.  
  
 Dans tous les cas, la résolution de ce problème consiste à ajouter la référence manquante au projet [!INCLUDE[vsprvs](../assembler/masm/includes/vsprvs_md.md)] proprement dit.
