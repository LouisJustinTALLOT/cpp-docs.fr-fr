# Assistant Projet ATL
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

La bibliothèque ATL \(Active Template Library\) est un ensemble de classes C\+\+ basées sur un modèle, qui facilitent la création des objets COM rapides et de petite taille.  L'Assistant Projet ATL crée un projet avec les structures nécessaires pour contenir les objets COM.  
  
## Vue d'ensemble  
 Cette page de l'Assistant décrit les [paramètres de l'application en cours pour le projet ATL](../../atl/reference/application-settings-atl-project-wizard.md) que vous créez.  Par défaut, le projet est défini avec les paramètres suivants :  
  
-   La bibliothèque de liens dynamiques spécifie que le serveur est une DLL et par conséquent un serveur in\-process.  
  
-   L'attribut spécifie que votre projet utilise des attributs.  
  
 Pour modifier ces paramètres par défaut, cliquez sur **Paramètres de l'application** dans la colonne de gauche de l'Assistant et effectuez les changements souhaités dans la page de l'Assistant Projet ATL.  
  
 Pour obtenir des informations sur les paramètres de projet par défaut, y compris le choix des jeux de caractères et les liaisons par défaut, consultez [Configurations des projets ATL par défaut](../../atl/reference/default-atl-project-configurations.md).  
  
 Après avoir créé votre projet ATL, vous pouvez y ajouter des objets ou des contrôles en utilisant les [Assistants Code](../../ide/adding-functionality-with-code-wizards-cpp.md) Visual C\+\+.  Ces Assistants Code vous permettent d'apporter les types d'améliorations suivants à un projet ATL de base :  
  
-   [Ajouter des objets et des contrôles à un projet ATL](../../atl/reference/adding-objects-and-controls-to-an-atl-project.md)  
  
-   [Ajouter une nouvelle interface à un projet ATL](../../atl/reference/adding-a-new-interface-in-an-atl-project.md)  
  
-   [Ajouter un composant COM\+ 1.0 à un projet ATL](../../atl/reference/adding-an-atl-com-plus-1-0-component.md)  
  
 En outre, vous pouvez effectuer les tâches suivantes lorsque vous créez et améliorez un projet ATL :  
  
-   [Assigner l'attribut noncreatable à un objet ATL](../../atl/reference/making-an-atl-object-noncreatable.md)  
  
-   [Optimiser le compilateur pour un projet ATL](../../atl/reference/specifying-compiler-optimization-for-an-atl-project.md)  
  
 Vous pouvez spécifier les propriétés du projet \(par exemple, [si le projet doit être lié statiquement au code CRT](../../atl/programming-with-atl-and-c-run-time-code.md)\) dans la page [Propriétés du projet](../../ide/general-property-page-project.md), ainsi que les [configurations de build](../Topic/Understanding%20Build%20Configurations.md) pour un projet ATL.  Pour plus d'informations, consultez [Utilisation de projets pour créer des applications](http://msdn.microsoft.com/fr-fr/3339fa90-bac2-4b95-8361-662a2e0e7dfe).
