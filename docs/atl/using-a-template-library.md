# <a name="using-a-template-library"></a>Utilisation d’une bibliothèque de modèles

Un modèle est un peu comme une macro. Comme avec une macro, l’appel d’un modèle entraîne son développement (avec la substitution de paramètre appropriée) au code que vous avez écrit. Toutefois, un modèle va plus loin pour permettre la création de nouvelles classes en fonction des types que vous transmettez en tant que paramètres. Ces nouvelles classes implémentent des méthodes de type sécurisé pour effectuer l’opération exprimée dans votre code de modèle.

Les bibliothèques de modèles comme ATL diffèrent des bibliothèques de classes C++ traditionnelles dans la mesure où elles sont généralement fournies uniquement en tant que code source (ou en tant que code source avec un peu, prenant en charge le temps d’exécution) et ne sont pas par nature hiérarchiques ou nécessairement hiérarchiques par nature. Au lieu d’effectuer une dérivation à partir d’une classe pour obtenir les fonctionnalités que vous souhaitez, vous instanciez une classe à partir d’un modèle.
