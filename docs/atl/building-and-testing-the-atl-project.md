# <a name="building-and-testing-the-atl-project"></a>Génération et test du projet ATL

Comme mentionné dans [insertion d’un contrôle Composite](../atl/inserting-a-composite-control.md), l’un des composants du projet initiales est une page HTML par défaut qui héberge votre nouveau contrôle composite. Une fois que vous avez terminé la modification du contrôle composite, cliquez sur **générer la Solution** ou **régénérer la Solution** à partir de la **Build** menu. Une fois que le projet est généré avec succès, chargez la page HTML, située dans le répertoire racine de votre projet de contrôle composite, dans Internet Explorer ou un autre navigateur et tester la fonctionnalité de votre contrôle.

Vous pouvez également tester votre contrôle composite à l’aide de l’outil de conteneur de Test, ou toute autre application qui peut héberger un contrôle ActiveX. Pour plus d’informations sur la façon d’accéder au conteneur de test, consultez la page [Test des propriétés et des événements avec le conteneur de test](../mfc/testing-properties-and-events-with-test-container.md) .
