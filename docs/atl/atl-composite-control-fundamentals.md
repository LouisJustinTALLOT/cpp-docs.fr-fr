# <a name="atl-composite-control-fundamentals"></a>Notions de base des contrôles composites ATL

Un contrôle composite est un type de contrôle ActiveX qui peut contenir (semblable à une boîte de dialogue) d’autres contrôles ActiveX ou contrôles Windows. Une fois le contrôle composite créé, il peut être inséré partout où un contrôle ActiveX peut être hébergé.

L’Assistant Projet ATL et la boîte de dialogue **Ajouter une classe** automatisent le processus de création et d’implémentation d’un projet de contrôle composite, comme le résultat de l’exécution de l’Assistant application pour créer une infrastructure d’application MFC. Le processus de développement se compose de cinq étapes :

- [Création d’un projet ATL](../atl/reference/creating-an-atl-project.md)

- [Insertion d’un contrôle composite](../atl/inserting-a-composite-control.md)

- [Modification du projet ATL](../atl/modifying-the-atl-project.md)

- [Ajout d’une fonctionnalité au contrôle composite](../atl/adding-functionality-to-the-composite-control.md)

- [Génération et test du projet ATL](../atl/building-and-testing-the-atl-project.md)
