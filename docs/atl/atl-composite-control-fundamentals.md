# <a name="atl-composite-control-fundamentals"></a>Notions de base des contrôles composites ATL

Un contrôle composite est un type de contrôle ActiveX qui peut contenir (semblable à une boîte de dialogue) autres contrôles ActiveX ou Windows. Une fois que le contrôle composite est créé, il peut être inséré n’importe où qu'un contrôle ActiveX peut être hébergé.

L’Assistant Projet ATL et **ajouter une classe** boîte de dialogue automatiser le processus de création et implémentation d’un projet de contrôle composite, similaire au résultat de l’exécution de l’Assistant Application pour créer une infrastructure d’application MFC. Le processus de développement se compose de cinq étapes :

- [Création d’un projet ATL](../atl/reference/creating-an-atl-project.md)

- [Insertion d’un contrôle composite](../atl/inserting-a-composite-control.md)

- [Modification du projet ATL](../atl/modifying-the-atl-project.md)

- [Ajout de fonctionnalités au contrôle composite](../atl/adding-functionality-to-the-composite-control.md)

- [Génération et test du projet ATL](../atl/building-and-testing-the-atl-project.md)
