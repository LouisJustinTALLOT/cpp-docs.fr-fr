# Notions de base des contr&#244;les composites ATL
[!INCLUDE[vs2017banner](../assembler/inline/includes/vs2017banner.md)]

Un contrôle composite est un type de contrôle ActiveX qui peut contenir \(semblable à une boîte de dialogue\) d'autres contrôles Windows ou contrôles ActiveX.  Une fois que le contrôle composite est généré, il peut être inséré n'importe où un contrôle ActiveX peut être hébergé.  
  
 La boîte de dialogue de l'Assistant Projet ATL et d' **Add Class** automatisent le processus de création et d'implémenter un projet de contrôle composite, semblable au résultat de l'exécution de l'Assistant Application pour créer une zone d'applications MFC.  Le processus de développement se compose de cinq étapes :  
  
-   [Créer un projet ATL](../atl/reference/creating-an-atl-project.md)  
  
-   [Insérer un contrôle composite](../atl/inserting-a-composite-control.md)  
  
-   [Modifier le projet ATL](../atl/modifying-the-atl-project.md)  
  
-   [Ajouter des fonctionnalités au contrôle composite](../atl/adding-functionality-to-the-composite-control.md)  
  
-   [Génération et test du projet ATL](../atl/building-and-testing-the-atl-project.md)
