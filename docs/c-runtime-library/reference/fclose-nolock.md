# _fclose_nolock
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

Ferme un flux de données sans verrouillage de thread.  
  
## Syntaxe  
  
```  
int _fclose_nolock(   
   FILE *stream   
);  
```  
  
#### Paramètres  
 `stream`  
 Pointeur vers la structure `FILE`.  
  
## Valeur de retour  
 `fclose` retourne 0 si le flux a été fermé avec succès.  Retourne  `EOF` pour indiquer une erreur.  
  
## Notes  
 Cette fonction est une version non verrouillée de`fclose`.  Il est identique à la différence qu'il n'est pas protégé d'une interférence de la part de les autres threads.  Elles peuvent être plus rapides, car elles n'entraînent pas la charge du verrouillage des autres threads.  Utilisez ces fonctions uniquement dans les contextes thread\-safe, tels que les applications à un seul thread ou lorsque la portée appelante gère déjà l'isolation des threads.
