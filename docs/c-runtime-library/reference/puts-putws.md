# <a name="puts-putws"></a>puts, _putws
Écrit une chaîne dans **stdout**.  
  
## <a name="syntax"></a>Syntaxe  
  
```  
  
      int puts(  
   const char *str   
);  
int _putws(  
   const wchar_t *str   
);  
```  
  
#### <a name="parameters"></a>Paramètres  
 `str`  
 Chaîne de sortie.  
  
## <a name="return-value"></a>Valeur de retour  
 Retourne une valeur non négative si l’opération aboutit. Si `puts` échoue, la valeur `EOF` est retournée ; si `_putws` échoue, la valeur retournée est **WEOF**. Si `str` est un pointeur Null, le gestionnaire de paramètre non valide est appelé, comme décrit dans [Validation de paramètre](../../c-runtime-library/parameter-validation.md). Si l’exécution est autorisée à se poursuivre, les fonctions affectent à `errno` la valeur `EINVAL` et retournent `EOF` ou **WEOF**.  
  
 Pour plus d’informations sur ces codes d’erreur et les autres, consultez [_doserrno, errno, _sys_errlist et _sys_nerr](../../c-runtime-library/errno-doserrno-sys-errlist-and-sys-nerr.md).  
  
## <a name="remarks"></a>Notes  
 La fonction `puts` écrit `str` dans le flux de sortie standard **stdout**, en remplaçant le caractère Null de fin ('\0') de la chaîne par un caractère de saut de ligne ('\n') dans le flux de sortie.  
  
 `_putws` est la version à caractères larges de `puts` ; les deux fonctions se comportent de la même manière si le flux est ouvert en mode ANSI. `puts` ne prend pas en charge la sortie vers un flux UNICODE pour l’instant.  
  
 Dans Windows 2000 et ultérieur, **_putwch** écrit des caractères Unicode en utilisant le paramètre CONSOLE LOCALE actif.

## <a name="example"></a>Exemple  
  
```  
// crt_puts.c  
/* This program uses puts to write a string to stdout.  
 */  
  
#include <stdio.h>  
  
int main( void )  
{  
   puts( "Hello world from puts!" );  
}  
```  
  
## <a name="output"></a>Sortie  
  
```  
Hello world from puts!  
```
