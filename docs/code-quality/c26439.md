# <a name="c26439-special_noexcept"></a>C26439 SPECIAL_NOEXCEPT

"Ce genre de fonction peut ne pas jeter. Déclarez-le «noexcept».

**Lignes directrices de base**de CMD : F.6 : Si votre fonction ne peut pas lancer, déclarez-la sans exemple

Certains types d’opérations ne devraient jamais causer d’exceptions. Leurs implémentations doivent être fiables et doivent gérer les conditions d’erreurs possibles avec grâce. Ils ne devraient jamais utiliser des exceptions pour indiquer l’échec. Cette règle signale les cas où de telles opérations ne sont pas explicitement marquées comme «noexcept» ce qui signifie qu’elles peuvent jeter des exceptions et ne peuvent pas transmettre des hypothèses sur leur fiabilité.

## <a name="remarks"></a>Notes

- Les types spéciaux d’opérations sont les suivants :
  - destructeurs;
  - constructeurs par défaut;
  - déplacer les constructeurs et déplacer les opérateurs d’affectation;
  - fonctions standard avec la sémantique de mouvement : std : : déplacer et std : : swap.
- Les spécificateurs non standard et désuets comme le lancer() ou le declspec (nothrow) ne sont pas équivalents à «noexcept».
- Les spécificateurs explicites noexcept (faux) et noexcept(true) sont respectés de manière appropriée.
- L’avertissement peut encore apparaître pour les opérations qui sont marquées comme constexpr. Cela pourrait changer dans les versions futures.
