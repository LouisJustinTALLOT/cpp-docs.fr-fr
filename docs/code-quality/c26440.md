# <a name="c26440-declare_noexcept"></a>C26440 DECLARE_NOEXCEPT

"La fonction peut être déclarée 'noexcept'."

**Lignes directrices de base**de CMD : F.6 : Si votre fonction ne peut pas lancer, déclarez-la sans exemple

Si le code n’est pas censé causer des exceptions, il doit être marqué en tant que tel en utilisant le spécificateur 'noexcept'. Cela aiderait à simplifier la gestion des erreurs du côté du code client, ainsi que de permettre au compilateur de faire des optimisations supplémentaires.

## <a name="remarks"></a>Notes

- Une fonction est considérée comme non-lancer si :
  - il n’a pas de déclarations explicites;
  - les appels de fonction dans son corps, le cas échéant, invoquent seulement des fonctions qui ne sont pas susceptibles de jeter : constexpr ou fonctions marquées avec n’importe quelle spécification d’exception qui implique un comportement non-jet (cela inclut quelques spécifications non standard).
- Les spécificateurs non standard et désuets comme le lancer() ou le declspec (nothrow) ne sont pas équivalents à «noexcept».
- Les spécificateurs explicites noexcept (faux) et noexcept(true) sont respectés de manière appropriée.
- Les fonctions marquées comme constexpr ne sont pas censées causer des exceptions et ne sont pas analysées.
- La règle s’applique également aux expressions lambda.
- La logique ne considère pas les appels récursifs comme potentiellement non-lancer. Cela pourrait changer à l’avenir.
