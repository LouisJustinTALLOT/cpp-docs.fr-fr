# <a name="c26485-no_array_to_pointer_decay"></a>C26485 NO_ARRAY_TO_POINTER_DECAY

> AVERTISSEMENT C26485 : expression'*Array-Name*' : aucune dégradation de tableau en pointeur (Bounds. 3).

## <a name="remarks"></a>Notes 

Comme [C26481](c26481.md), cette vérification permet d’appliquer la règle de [C++ Core Guidelines](https://github.com/isocpp/CppCoreGuidelines/blob/master/CppCoreGuidelines.md) [I. 13](https://github.com/isocpp/CppCoreGuidelines/blob/master/CppCoreGuidelines.md#Ri-array): *ne transmettez pas un tableau en tant que pointeur unique*. La règle détecte les endroits où les informations de type tableau statiques sont perdues d’une atténuation à un pointeur brut. Les `zstring` types `czstring` et ne sont pas exclus.

C26481 et C26485 proviennent des règles du [profil de sécurité des limites](https://github.com/isocpp/CppCoreGuidelines/blob/master/CppCoreGuidelines.md#SS-bounds) . Ces règles ont été implémentées dans la première version de l’outil de vérification de C++ Core Guidelines. Elles s’appliquent à la catégorie des pointeurs bruts, car elles permettent d’éviter une utilisation non sécurisée des pointeurs bruts.

## <a name="example"></a> Exemple

Cet exemple génère deux avertissements pour la désintégration de tableau à pointeur dans l' `memcpy`appel à.

```cpp
// c26485_bad.cpp
// compile using:
// set Esp.Extensions=CppCoreCheck.dll
// cl /W4 /EHsc /permissive- /analyze /analyze:plugin EspXEngine.dll /analyze:ruleset "%VSINSTALLDIR%\Team Tools\Static Analysis Tools\Rule Sets\CppCoreCheckBoundsRules.ruleset" c26485_bad.cpp
#include <cstring>
constexpr int array_length = 10;

int main() noexcept
{
    int const from_array[array_length] = { 4, 3, 2, 1, 0, 9, 8, 7, 6, 5 };
    int to_array[array_length] = {};

    if (nullptr != memcpy(to_array, from_array, sizeof(int) * array_length))
        return 0;
    return 1;
}
```

Pour résoudre ce problème, évitez les appels qui prennent des paramètres de pointeur, mais ne gérez pas les informations relatives aux limites. L’utilisation de ces fonctions est souvent sujette aux erreurs. Préférer les appels de la bibliothèque standard C++ aux fonctions de la bibliothèque Runtime C. Envisagez d’utiliser `gsl::span` ou `std::vector` dans vos propres fonctions. Un cast explicite vers le type de pointeur en atténuation empêche l’avertissement, mais il n’empêche pas le code de bogue.
