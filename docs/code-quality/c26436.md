# <a name="c26436-need_virtual_dtor"></a>C26436 NEED_VIRTUAL_DTOR

« Le type avec une fonction virtuelle a besoin d’un destructeur virtuel public ou protégé non virtual. »

**Lignes directrices de base**: C.35 : Un destructeur de classe de base doit être public et virtuel, soit protégé et nonvirtual

Si une classe définit une fonction virtuelle, elle devient polymorphe, ce qui implique que les classes dérivées peuvent changer son comportement, y compris la gestion des ressources et la logique de destruction. Étant donné que le code client peut appeler des types polymorphes via des pointeurs pour les classes de base, il n’y a aucun moyen qu’un client puisse choisir explicitement quel comportement est approprié sans downcasting. Pour s’assurer que les ressources sont gérées de façon cohérente et que la destruction se produit selon les règles du type réel, il est recommandé de définir un destructeur virtuel public. Si la hiérarchie de type est conçue pour interdire le code client pour détruire directement les objets, les destructeurs doivent être définis comme non virtuels protégés.

## <a name="remarks"></a>Notes

- L’avertissement apparaît sur la première définition de fonction virtuelle d’un type (il peut être un destructeur virtuel s’il n’est pas public), une fois par type.
  - Étant donné que la définition peut être placée séparément de la déclaration, elle ne peut pas toujours avoir l’un des spécificateurs virtuels. Mais l’avertissement est toujours valable - il vérifie la «virtualité» réelle d’une fonction.
