# <a name="c26433-override_explicitly"></a>C26433 OVERRIDE_EXPLICITLY

La fonction doit être marquée par`override`

## <a name="c-core-guidelines"></a>Lignes directrices de base de CMD

[C.128 : Les fonctions virtuelles doivent spécifier exactement une fonction virtuelle, de remplacement ou de](https://github.com/isocpp/CppCoreGuidelines/blob/master/CppCoreGuidelines.md)

Il n’est pas nécessaire par le compilateur d’affirmer clairement qu’une fonction virtuelle l’emporte sur sa base. Ne pas `override` spécifier peut causer des problèmes subtils pendant la maintenance si la spécification virtuelle change jamais dans la hiérarchie de la classe. Il abaisse également la lisibilité et rend le comportement polymorphe d’une interface moins évident. Si une fonction est `override`clairement marquée comme , il permet au compilateur de vérifier la cohérence de l’interface et aider à repérer les problèmes avant qu’ils se manifestent au moment de l’exécution.

## <a name="notes"></a>Notes

1. Cette règle ne s’applique pas aux destructeurs. Les destructeurs ont leurs propres spécificités de virtualité.
1. La règle ne signale pas `final`les fonctions explicitement marquées comme , qui est elle-même une variété spéciale de spécificateur virtuel.
1. Les avertissements apparaissent sur les définitions de fonction, pas les déclarations. Il peut être déroutant, puisque les définitions n’ont pas de spécificateurs virtuels, mais l’avertissement est toujours correct.

## <a name="example--implicit-overriding"></a>Exemple : Dépassement implicite

```cpp
class Shape {
public:
    virtual void Draw() = 0;
    // ...
};

class Ellipse : public Shape {
public:
    void Draw() { // C26433
        //...
    }
};
```
