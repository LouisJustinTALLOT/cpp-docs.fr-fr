# <a name="c26443-no_explicit_dtor_override"></a>C26443 NO_EXPLICIT_DTOR_OVERRIDE

« Le destructeur dominant ne devrait pas utiliser des spécificateurs explicites de « remplacement » ou « virtuels ».

## <a name="c-core-guidelines"></a>Lignes directrices de base de CMD

[C.128 : Les fonctions virtuelles doivent spécifier exactement une fonction virtuelle, de remplacement ou finale.](https://github.com/isocpp/CppCoreGuidelines/blob/master/CppCoreGuidelines.md)

Le consensus actuel sur les Lignes directrices fondamentales est d’exclure les destructeurs de la recommandation « dérogation explicitement ».

## <a name="notes"></a>Notes

- La règle signale les déstructeurs dominants qui utilisent explicitement des spécificateurs « virtuels » ou « dérogations ».
- Les destructeurs peuvent toujours utiliser le spécificateur « final » en raison de sa sémantique spéciale.
- Les avertissements apparaissent sur les définitions de fonction, pas les déclarations. Il peut être déroutant, puisque les définitions n’ont pas de spécificateurs virtuels, mais l’avertissement est toujours approprié.

## <a name="example-explicit-override"></a>Exemple : « remplacement » explicite

```cpp
class Transaction {
public:
    virtual ~Transaction();
    // ...
};

class DistributedTransaction : public Transaction {
public:
    ~DistributedTransaction() override { // C26443
        // ...
    }
};
```
