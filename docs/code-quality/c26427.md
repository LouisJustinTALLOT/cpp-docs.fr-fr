# <a name="c26427-no_global_init_externs"></a>C26427 NO_GLOBAL_INIT_EXTERNS

« Global initializer accesse l’objet extern. »

**Lignes directrices de base**: I.22 : Éviter une initialisation complexe des objets mondiaux

Les objets globaux peuvent être parasités dans un ordre incohérent ou indéfini, ce qui signifie que l’interdépendance entre eux est risquée et doit être évitée. Cette ligne directrice s’applique lorsque les initialisateurs se `extern`réfèrent à un autre objet qui est considéré comme .

## <a name="remarks"></a>Notes

Un objet est `extern` réputé s’il est conforme aux règles suivantes :

- c’est une variable `extern` globale marquée avec un spécificateur ou c’est un membre statique d’une classe;
- ce n’est pas dans un espace nom anonyme;
- il n’est `const`pas marqué comme ;
- Les membres de la classe statique sont considérés comme globaux, de sorte que leurs initialisateurs sont également vérifiés.

## <a name="example"></a>Exemple

vérification de version externe

```cpp
// api.cpp
int api_version = API_DEFAULT_VERSION; // Assume it can change at run time, hence non-const.

// client.cpp
extern int api_version;
bool is_legacy_mode = api_version <= API_LEGACY_VERSION; // C26427, also stale value
```

vérification de version externe - rendue plus fiable

```cpp
// api.cpp
int api_version = API_DEFAULT_VERSION; // Assume it can change at run time, hence non-const.

// client.cpp
extern int api_version;
bool is_legacy_mode() noexcept
{
    return api_version <= API_LEGACY_VERSION;
}
```
