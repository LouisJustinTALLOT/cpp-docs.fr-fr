# <a name="c26474-no_implicit_cast"></a>C26474 NO_IMPLICIT_CAST

"Ne pas jeter entre les types de pointeurs lorsque la conversion pourrait être implicite."

**Lignes directrices de base**de CMD : Type.1 : Évitez les moulages

Dans certains cas, les lancers implicites entre les types de pointeurs peuvent être effectués en toute sécurité et n’exigent pas que l’utilisateur écrive une expression de fonte spécifique. Cette règle détecte des cas de casting inutile qui peuvent être supprimés.

## <a name="remarks"></a>Notes

- La règle ID est un peu trompeuse: il doit être interprété comme "casting implicite n’est pas utilisé là où il est acceptable".
  - La règle s’applique uniquement aux pointeurs et vérifie les moulages statiques et les moulages réinterprétés.
  - Les cas suivants sont des conversions de pointeurs acceptables qui ne devraient pas utiliser des expressions explicites de fonte :
  - conversion à nullptr_t;
  - conversion en vide;
  - conversion du type dérivé à sa base.

## <a name="example"></a>Exemple

conversion inutile cache erreur logique

```cpp
template<class T>
bool register_buffer(T buffer) {
    auto p = reinterpret_cast<void*>(buffer); // C26474, also 26490 NO_REINTERPRET_CAST
    return buffers_.insert(p).second;
}

void merge_bytes(std::uint8_t *left, std::uint8_t *right)
{
    if (left && register_buffer(*left)) { // Unintended dereference!
        // ...
        if (right && register_buffer(right)) {
            // ...
        }
    }
}
```

conversion inutile cache erreur logique - retravaillé

```cpp
// ...
template<class T>
bool register_buffer(T *buffer) {
    auto p = buffer;
    return buffers_.insert(p).second;
}
// ...
```
