# <a name="arithmetic-overflow-operator-operation-causes-overflow-at-compile-time-use-a-wider-type-to-store-the-operands"></a>Débordement arithmétique : l’opération « % d’opérateur » provoque un débordement au moment de la compilation. Utilisez un type plus large pour stocker les opérands

Cet avertissement indique qu’une opération arithmétique était manifestement déficiteuse au moment de la compilation. Cela peut être affirmé lorsque les opérands sont tous compiler des constantes de temps.  Actuellement, nous vérifions les opérations de quart de travail, de multiplication, d’ajout et de soustraction gauches pour de tels débordements.

Remarque : C4307 est un contrôle similaire dans le compilateur Microsoft C.

## <a name="example-1"></a>Exemple 1

```cpp
int multiply()
{
    const int a = INT_MAX;
    const int b = 2;
    int c = a * b; // C26450 reported here
    return c;
}
```

Pour corriger cet avertissement, utilisez le code suivant.

```cpp
long long multiply()
{
    const int a = INT_MAX;
    const int b = 2;
    long long c = (long long)a * b; // OK
    return c;
}
```
