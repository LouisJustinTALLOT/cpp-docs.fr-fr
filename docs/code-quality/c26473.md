# <a name="c26473-no_identity_cast"></a>C26473 NO_IDENTITY_CAST

"Ne pas jeter entre les types de pointeurs où le type de source et le type de cible sont les mêmes."

**Lignes directrices de base**de CMD : Type.1 : Évitez les moulages

Cette règle permet d’éliminer les moulages inutiles ou suspects. Évidemment, lorsque le type est converti en lui-même, une telle conversion est inefficace, mais le fait que le casting est utilisé peut indiquer un problème de conception subtile ou un potentiel de régression si les types changent à l’avenir. Il est toujours plus sûr d’utiliser le moins de moulages possible.

## <a name="remarks"></a>Notes

- Cette règle est implémentée pour les moulages statiques et réinterprétés et ne vérifie que les types de pointeurs.

## <a name="example"></a>Exemple

dangereusement générique lookup

```cpp
gsl::span<server> servers_;

template<class T>
server* resolve_server(T tag) noexcept {
    auto p = reinterpret_cast<server*>(tag); // C26473, also 26490 NO_REINTERPRET_CAST
    return p >= &(*servers_.begin()) && p < &(*servers_.end()) ? p : nullptr;
}

void promote(server *s, int index) noexcept {
    auto s0 = resolve_server(s);
    auto s1 = resolve_server(index);
   if (s0 && s1)
        std::swap(s0, s1);
}
```

recherche dangereusement générique - retravaillé

```cpp
// ...
server* resolve_server(server *p) noexcept {
    return p >= &(*servers_.begin()) && p < &(*servers_.end()) ? p : nullptr;
}

server* resolve_server(ptrdiff_t i) noexcept {
    return !servers_.empty() && i >= 0 && i < servers_.size() ? &servers_[i] : nullptr;
}
// ...
```
