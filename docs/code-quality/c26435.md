# <a name="c26435-single_virtual_specification"></a>C26435 SINGLE_VIRTUAL_SPECIFICATION

«La fonction doit spécifier exactement celle de «virtuel», «remplacement», ou «final».

## <a name="c-core-guidelines"></a>Lignes directrices de base de CMD

[C.128 : Les fonctions virtuelles doivent spécifier exactement une fonction virtuelle, de remplacement ou de](https://github.com/isocpp/CppCoreGuidelines/blob/master/CppCoreGuidelines.md)

Pour améliorer la lisibilité, le type de comportement virtuel doit être énoncé clairement et sans redondance inutile. Même si plusieurs spécificateurs virtuels peuvent être utilisés simultanément, il est préférable de spécifier un à la fois pour souligner l’aspect le plus important du comportement virtuel. L’ordre d’importance suivant est évident :

- fonction virtuelle ordinaire;
- fonction virtuelle qui l’emporte explicitement sur sa base;
- fonction virtuelle qui l’emporte sur sa base et fournit la mise en œuvre finale dans la chaîne d’héritage actuelle.

## <a name="notes"></a>Notes

- Cette règle saute les destructeurs car ils ont des règles spéciales concernant la virtualité.
- Les avertissements apparaissent sur les définitions de fonction, pas les déclarations. Il peut être déroutant, puisque les définitions n’ont pas de spécificateurs virtuels, mais l’avertissement est toujours approprié.

## <a name="example-redundant-specifier"></a>Exemple : Spécificateur redondant

```cpp
class Ellipse : public Shape {
public:
    void Draw() override {
        //...
    }
};

class Circle : public Ellipse {
public:
    void Draw() override final { // C26435, only 'final' is necessary.
        //...
    }
};
```
