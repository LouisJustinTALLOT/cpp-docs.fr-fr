# <a name="afxdll-versions"></a>Versions d'AFXDLL
Au lieu de générer votre application en liant statiquement aux bibliothèques de code objet MFC, vous pouvez générer votre application pour utiliser une des bibliothèques AFXDLL, qui contiennent des MFC dans une DLL que plusieurs applications en cours d’exécution peuvent partager. Pour une table de noms AFXDLL, consultez [DLL : Conventions de nom](../build/naming-conventions-for-mfc-dlls.md).  
  
> [!NOTE]
>  Par défaut, l’Assistant Application MFC crée un projet AFXDLL. Pour utiliser la liaison statique du code MFC à la place, définissez le **utiliser les MFC dans une bibliothèque statique** option dans l’Assistant Application MFC. La liaison statique n’est pas disponible dans l’Édition Standard de Visual C++.
