# <a name="cmdiframewndex-class"></a>Classe CMDIFrameWndEx
Étend les fonctionnalités de [CMDIFrameWnd](../../mfc/reference/cframewnd-class.md), une fenêtre frame de Windows Interface MDI (Multiple Document).  
  
## <a name="syntax"></a>Syntaxe  
  
```  
class CMDIFrameWndEx : public CMDIFrameWnd  
```

##  <a name="addpane"></a>CMDIFrameWndEx::AddPane  
 Enregistre un volet avec le Gestionnaire d’ancrage.  
  
```  
BOOL AddPane(
    CBasePane* pControlBar,  
    BOOL bTail=TRUE);
```  
  
### <a name="parameters"></a>Paramètres  
 [in] `pControlBar`  
 Pointeur vers le volet à inscrire.  
  
 [in] `bTail`  
 Spécifie s’il faut ajouter ce volet à la fin de la liste.  
  
### <a name="return-value"></a>Valeur de retour  
 Retourne une valeur différente de zéro si le volet est correctement inscrit. Retourne 0 si le volet est déjà inscrit avec le Gestionnaire d’ancrage.  
  
### <a name="remarks"></a>Remarques  
 Chaque volet doit être enregistré avec le [CDockingManager classe](../../mfc/reference/cdockingmanager-class.md) avant que peut prendre une partie de la mise en page d’accueil. Utilisez cette méthode pour informer le Gestionnaire d’ancrage que vous souhaitez ancrer un volet spécifique. Une fois ce volet est enregistré, le Gestionnaire d’ancrage s’aligne en fonction de son paramètre d’alignement et de position dans la liste de volets maintenue par le Gestionnaire d’ancrage.

##  <a name="aremditabs"></a>CMDIFrameWndEx::AreMDITabs  
 Détermine si la fonctionnalité d’onglets MDI ou les groupes avec onglet MDI est activé.  
  
```  
BOOL AreMDITabs(int* pnMDITabsType=NULL) const;  
```  
  
### <a name="parameters"></a>Paramètres  
 [out] `pnMDITabsType`  
 Pointeur vers une variable de type entier qui indique les fonctionnalités qui sont activées :  
  
-   0 : toutes les fonctionnalités sont désactivées.  
  
-   1 : les onglets MDI est activé.  
  
-   2 : groupes avec onglet MDI est activé.  
  
### <a name="return-value"></a>Valeur de retour  
 `Returns TRUE`Si les onglets MDI ou MDI avec onglets groupes sont activé.  
  
 `Returns FALSE`Si aucune des fonctionnalités ci-dessus est activé.  
  
### <a name="remarks"></a>Notes  
 Utilisez cette fonction pour déterminer si les onglets MDI ou MDI avec onglets groupes est activée pour la fenêtre frame. Utilisez [CMDIFrameWndEx::EnableMDITabs](#enablemditabs) pour activer ou désactiver la fonctionnalité onglets MDI.  
  
 Utilisez [CMDIFrameWndEx::EnableMDITabbedGroups](#enablemditabbedgroups) pour activer ou désactiver la fonctionnalité de groupes avec onglet MDI.

##  <a name="savemdistate"></a>CMDIFrameWndEx::SaveMDIState  
 Enregistre la disposition actuelle des groupes d’onglets MDI et la liste des documents précédemment ouverts.  
  
```  
virtual BOOL SaveMDIState(LPCTSTR lpszProfileName);
```  
  
### <a name="parameters"></a>Paramètres  
 [in] `lpszProfileName`  
 Spécifie le nom du profil.  
  
### <a name="return-value"></a>Valeur de retour  
 `TRUE`Si l’enregistrement a réussi ; `FALSE` en cas d’échec de l’enregistrement.  
  
### <a name="remarks"></a>Notes  
 Pour charger ou enregistrer l’état des onglets MDI et des groupes et la liste des documents ouverts, procédez comme suit :  
  
-   Appelez `SaveMDIState` lorsque le frame principal est en cours de fermeture  
  
-   Appelez [CMDIFrameWndEx::LoadMDIState](#loadmdistate) lorsque le frame principal est créé. L’emplacement recommandé pour cet appel est avant le frame principal s’affiche pour la première fois.  
  
-   Appelez `CWinAppEx::EnableLoadWindowPlacement(FALSE);` avant`pMainFrame->LoadFrame (IDR_MAINFRAME);`  
  
-   Appelez `CWinAppEx::ReloadWindowPlacement``(pMainFrame)` après `LoadMDIState` pour afficher le frame principal à la position qui a été stockée dans le Registre.  
  
-   Substituer `GetDocumentName` dans le `CMDIChildWndEx`-classe dérivée si votre application affiche des documents qui ne sont pas stockées sous forme de fichiers. La chaîne retournée sera enregistrée dans le Registre sous un identificateur de document. Pour plus d’informations, consultez [CMDIChildWndEx::GetDocumentName](../../mfc/reference/cmdichildwndex-class.md#getdocumentname).  
  
-   Substituer [CMDIFrameWndEx::CreateDocumentWindow](#createdocumentwindow) pour créer des documents correctement lorsqu’elles sont chargées à partir du Registre. Le paramètre `CreateDocumentWindow` est la chaîne qui `GetDocumentName` renvoyée précédemment.
