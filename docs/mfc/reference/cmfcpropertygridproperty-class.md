## <a name="cmfcpropertygridpropertyaddoption"></a><a name="addoption"></a>CMFCPropertyGridProperty::AddOption

Ajoute un nouvel élément de liste à un contrôle de liste de propriété.

```
BOOL AddOption(
    LPCTSTR lpszOption,
    BOOL bInsertUnique=TRUE);
```

### <a name="parameters"></a>Paramètres

*lpszOption*<br/>
[dans] L’élément de liste (option) à ajouter.

*bInsertUnique*<br/>
[dans] VRAI pour ajouter l’élément de liste seulement s’il n’existe pas déjà; autrement, FALSE. La valeur par défaut est TRUE.

### <a name="return-value"></a>Valeur de retour

VRAI, ce qui signifie que l’élément de liste est ajouté. Sinon, FALSE, ce qui signifie que l’élément de liste n’est pas ajouté parce que le paramètre *bInsertUnique* est VRAI et l’élément de liste spécifié par le paramètre *lpszOption* existe déjà.

## <a name="cmfcpropertygridpropertyaddsubitem"></a><a name="addsubitem"></a>CMFCPropertyGridProperty::AddSubItem

Ajoute un article pour enfant à une propriété.

```
BOOL AddSubItem(CMFCPropertyGridProperty* pProp);
```

### <a name="parameters"></a>Paramètres

*pPropulse*<br/>
[dans] Pointeur vers une propriété à ajouter.

### <a name="return-value"></a>Valeur de retour

VRAI si la propriété spécifiée est ajoutée avec succès comme une propriété pour enfants. FALSE si la propriété n’est pas ajoutée parce qu’elle se produit déjà dans la propriété mère.

### <a name="remarks"></a>Notes

Utilisez cette méthode pour créer une liste hiérarchique des propriétés parent et enfant. Une fois qu’une propriété pour enfants est ajoutée, la propriété mère affiche automatiquement un contrôle de boîte d’extension qui est désigné par un signe positif ( ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' ' Lorsque l’utilisateur clique sur le signe positif, la propriété mère s’élargit et affiche tous les articles de propriété pour enfants.

## <a name="cmfcpropertygridpropertyadjustbuttonrect"></a><a name="adjustbuttonrect"></a>CMFCPropertyGridProperty::AdjustButtonRect

Appelé par le contrôle de la liste de propriété parente pour dire à une propriété de resize le rectangle de délimitation d’un bouton intégré.

```
virtual void AdjustButtonRect();
```

### <a name="remarks"></a>Notes

Par défaut, cette méthode :

- Ajuste la largeur du bouton égale à la hauteur du bouton plus 3 pixels.

- Déplace le rectangle de délimitation du bouton vers le bord droit de la propriété

- Déplace le bouton 1 pixel au-dessous du bord supérieur de la propriété.

## <a name="cmfcpropertygridpropertyadjustinplaceeditrect"></a><a name="adjustinplaceeditrect"></a>CMFCPropertyGridProperty::AdjustInPlaceEditRect

Récupère les limites de la boîte de texte et le contrôle optionnel du bouton de rotation qui sont utilisés pour définir une valeur de propriété.

```
virtual void AdjustInPlaceEditRect(
    CRect& rectEdit,
    CRect& rectSpin);
```

### <a name="parameters"></a>Paramètres

*rectEdit rectEdit*<br/>
[out] Lorsque cette méthode revient, un rectangle qui spécifie les limites de la boîte de texte pour la valeur de la propriété.

*rectSpin*<br/>
[out] Lorsque cette méthode revient, un rectangle qui spécifie les limites du contrôle du bouton de rotation pour la valeur de la propriété. Ou, si la propriété ne prend pas en charge un bouton de rotation, un rectangle vide.

### <a name="remarks"></a>Notes

La zone de valeur d’une propriété se compose d’une boîte de texte et peut-être d’un *bouton d’option,* comme un contrôle de bouton de spin. Cette méthode calcule les dimensions de la boîte de texte et du bouton d’option, puis renvoie ces valeurs dans les paramètres spécifiés.

## <a name="cmfcpropertygridpropertyallowedit"></a><a name="allowedit"></a>CMFCPropertyGridProperty::AllowEdit

Rend une propriété soit modifiable ou lue uniquement.

```cpp
void AllowEdit(BOOL bAllow=TRUE);
```

### <a name="parameters"></a>Paramètres
