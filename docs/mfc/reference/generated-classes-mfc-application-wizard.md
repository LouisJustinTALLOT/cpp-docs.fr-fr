# Classes g&#233;n&#233;r&#233;es, Assistant Application MFC
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

Cette rubrique répertorie les noms des classes de base et des fichiers générés par votre projet.  Par défaut, les noms se basent sur le nom du projet que vous avez spécifié dans la **boîte de dialogue Nouveau projet**.  Vous pouvez changer la majorité de ces noms, comme décrit ci\-après :  
  
 **Classes générées**  
 Noms des classes créées pour le projet.  Par défaut, les noms se basent sur le nom du projet.  Le projet de MFC par défaut crée la classe d'affichage en cours, une classe C de vue*ProjName*, une classe C de documents*ProjName*, la classe de doc en cours, *ProjName*une classe de CMainFrame, et une classe de CChildFrame.  Toutes les autres valeurs figurant dans cette page contiennent des informations concernant la classe actuellement sélectionnée dans la liste **Classes générées**.  
  
 Pour changer un nom de classe, utilisez la zone de texte **Nom de la classe**.  
  
 **Nom de classe**  
 Nom de la classe actuellement sélectionnée dans la liste **Classes générées**.  Si la zone est active, vous pouvez changer le nom de la classe.  Lorsque vous changez le focus de la zone **Nom de la classe**, tout changement effectué au niveau du nom de classe sélectionné apparaît dans la liste **Classes générées**.  
  
 **fichier .h**  
 Nom du fichier d'en\-tête de la classe actuellement sélectionnée dans la liste **Classes générées**.  Si la zone de texte est active, vous pouvez changer le nom du fichier d'en\-tête.  
  
 **Classe de base**  
 Nom de la classe de base de la classe actuellement sélectionnée dans la liste **Classes générées**.  Si la zone est active, vous pouvez sélectionner dans la liste une autre classe pour la classe de base.  
  
 **fichier .cpp**  
 Nom du fichier de code source associé à la classe sélectionnée.  Si la zone de texte est active, vous pouvez changer le nom du fichier d'implémentation.
