# <a name="cmfcribbonpanel-class"></a>Classe CMFCRibbonPanel

Implémente un panneau qui contient un jeu d'éléments de ruban. Lorsque le panneau est dessiné, il affiche le plus d'éléments possible, selon la taille du panneau.

Pour plus de détails, consultez le code source situé dans le dossier **VC\\\\atlmfc src\\mfc** de votre installation Visual Studio.

## <a name="syntax"></a>Syntaxe

```
class CMFCRibbonPanel : public CObject
```

## <a name="remarks"></a>Notes

Les panneaux de ruban sont des regroupements logiques de tâches connexes que vous créez dans les catégories de ruban. Au fur et à mesure que la taille du ruban change, la disposition du panneau s’ajuste automatiquement pour afficher autant d’éléments que possible.

Vous pouvez obtenir un panneau ruban qui est contenu dans une catégorie de ruban en appelant la [cmFCRibbonCategory::GetPanel](../../mfc/reference/cmfcribboncategory-class.md#getpanel) méthode.

## <a name="cmfcribbonpaneladdtoolbar"></a><a name="addtoolbar"></a>CMFCRibbonPanel::AddToolBar

Ajoute une barre d’outils au panneau ruban.

```
CMFCRibbonButtonsGroup* AddToolBar(
UINT uiToolbarResID,
UINT uiColdResID = 0,
UINT uiHotResID = 0,
UINT uiDisabledResID = 0);
```

### <a name="parameters"></a>Paramètres

*uiToolbarResID*<br/>
[dans] Spécifie l’ID de ressources de la barre d’outils à ajouter.

*uiColdResID*<br/>
[dans] Spécifie l’id de ressources des images froides de la barre d’outils.

*uiHotResID*<br/>
[dans] Spécifie l’id de ressources des images chaudes de la barre d’outils.

*uiDisabledResID*<br/>
[dans] Spécifie l’id de ressources des images désactivées de la barre d’outils.

### <a name="return-value"></a>Valeur de retour

Appelez cette méthode pour ajouter une barre d’outils au panneau ruban. La barre d’outils sera ajoutée à côté de l’élément ruban ajouté par l’appel précédent à [CMFCRibbonPanel::Ajouter](#add).

### <a name="remarks"></a>Notes

Pour plus d’informations sur les barres d’outils, les images chaudes, les images froides et les images désactivées, voir [CMFCToolBar Class](../../mfc/reference/cmfctoolbar-class.md).

```
CRuntimeClass* pRTC);
```

### <a name="parameters"></a>Paramètres

*uiCmdID*<br/>
[dans] Spécifie l’ID de commande de l’élément ruban à ajouter.

*Cfrp*<br/>
[dans, dehors] Un pointeur pour les informations de classe de temps d’exécution associés à l’élément ruban qui est ajouté au panneau de ruban.

### <a name="return-value"></a>Valeur de retour

L’élément ruban qui a été créé en utilisant les informations spécifiées de classe de temps d’exécution.

### <a name="remarks"></a>Notes

Si vous souhaitez ajouter un élément personnalisé (par exemple, un bouton de couleur) au panneau ruban, vous devez spécifier les informations de classe de temps d’exécution de l’élément personnalisé. Le ruban stocke ces informations, crée l’élément personnalisé et remplace un élément existant situé par l’ID de commande spécifié. Il renvoie ensuite un pointeur à l’élément nouvellement créé.

### <a name="example"></a>Exemple

L’exemple suivant montre `SetElementRTCByID` comment utiliser la méthode :

```

// Load and add toolbar with standard buttons. This toolbar
// should display a custom color button with id ID_CHAR_COLOR:

pPanel->AddToolBar(IDR_MAINFRAME,
    IDB_MAINFRAME256);

CMFCRibbonColorButton* pColorButton =
(CMFCRibbonColorButton*)pPanel->SetElementRTCByID(
ID_CHAR_COLOR,
    RUNTIME_CLASS (CMFCRibbonColorButton));

// SetElementRTCByID sets runtime class and returns a pointer
// to the newly created custom button,
    which can be set up immediately:
pColorButton->EnableAutomaticButton(_T("Automatic"),
    RGB (0,
    0,
    0));
```
