# <a name="cmdichildwndex-class"></a>CMDIChildWndEx, classe

La `CMDIChildWndEx` classe fournit la fonctionnalité d’une fenêtre pour enfants Windows à interface de plusieurs documents (MDI). Il étend la fonctionnalité de [CMDIChildWnd Class](../../mfc/reference/cmdichildwnd-class.md). L'infrastructure requiert cette classe lorsqu'une application MDI utilise certaines classes MFC.

Pour plus de détails, consultez le code source situé dans le dossier **VC\\\\atlmfc src\\mfc** de votre installation Visual Studio.

## <a name="syntax"></a>Syntaxe

```
class CMDIChildWndEx : public CMDIChildWnd
```

## <a name="cmdichildwndexdockpaneleftof"></a><a name="dockpaneleftof"></a>CMDIChildWndEx::DockPaneLeftOf

```

    CPane* pBar,
    CPane* pLeftOf);
```

### <a name="parameters"></a>Paramètres

*pBar (pBar)*<br/>
Un pointeur à la vitre qui doit être amarré.

*pLeftOf*<br/>

Appelez cette méthode lorsque vous voulez amarrer plusieurs volets dans l’ordre prédéfini.

Un document est ce que la fenêtre de l’enfant MDI affiche. En général, la fenêtre affiche les données qui sont chargées ou enregistrées à un fichier. Par conséquent, le nom du document est le nom du fichier. La mise `GetDocumentName` en œuvre `CDocument::GetPathName`par défaut des retours d’une chaîne obtenue à partir de .

Si la fenêtre affiche un document qui n’est pas chargé à partir d’un fichier, remplacez cette méthode dans une classe dérivée et retournez un identifiant de document unique.

`GetDocumentName`est appelé par le cadre quand il sauve l’état de tous les documents ouverts. La chaîne retournée est écrite au registre.

Lorsque le cadre est la restauration de l’état plus tard, le nom du document est lu à partir du registre et passé à [CMDIFrameWndEx::CreateDocumentWindow](../../mfc/reference/cmdiframewndex-class.md#createdocumentwindow). Remplacez cette méthode dans une classe [DÉRIVÉe CMDIFrameWndEx](../../mfc/reference/cmdiframewndex-class.md)et créez ou ouvrez un document qui porte ce nom et lue dans le fichier qui porte ce nom. Si le document n’est pas basé sur un fichier, créez le document en fonction de l’identifiant du document lui-même. Vous ne devez faire les actions précédentes que si vous avez l’intention d’enregistrer et de restaurer des documents.
