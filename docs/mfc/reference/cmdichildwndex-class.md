# <a name="cmdichildwndex-class"></a>CMDIChildWndEx, classe

La `CMDIChildWndEx` classe fournit les fonctionnalités d’une fenêtre enfant MDI (multiple document interface) Windows. Il étend les fonctionnalités de la [classe CMDIChildWnd](../../mfc/reference/cmdichildwnd-class.md). L'infrastructure requiert cette classe lorsqu'une application MDI utilise certaines classes MFC.

Pour plus d’informations, consultez le code source situé dans le dossier **VC\\ATLMFC\\SRC\\MFC** de votre installation de Visual Studio.

## <a name="syntax"></a>Syntaxe

```
class CMDIChildWndEx : public CMDIChildWnd
```

##  <a name="getdocumentname"></a>  CMDIChildWndEx::GetDocumentName

Retourne le nom du document affiché dans la fenêtre enfant MDI.

```
virtual LPCTSTR GetDocumentName(CObject** pObj);
```

### <a name="return-value"></a>Valeur de retour

Pointeur vers une chaîne qui contient le nom d’un document.

### <a name="remarks"></a>Notes

Un document correspond à ce que la fenêtre enfant MDI affiche. En règle générale, la fenêtre affiche les données qui sont chargées à partir de ou enregistrées dans un fichier. Par conséquent, le nom du document est le nom du fichier. L’implémentation par défaut `GetDocumentName` de retourne une chaîne obtenue `CDocument::GetPathName`à partir de.

Si la fenêtre affiche un document qui n’est pas chargé à partir d’un fichier, substituez cette méthode dans une classe dérivée et retournez un identificateur de document unique.

`GetDocumentName`est appelé par le Framework lorsqu’il enregistre l’état de tous les documents ouverts. La chaîne retournée est écrite dans le registre.

Quand le Framework restaure l’État ultérieurement, le nom du document est lu à partir du Registre et transmis à [CMDIFrameWndEx:: CreateDocumentWindow](../../mfc/reference/cmdiframewndex-class.md#createdocumentwindow). Substituez cette méthode dans une classe dérivée de [CMDIFrameWndEx](../../mfc/reference/cmdiframewndex-class.md)et créez ou ouvrez un document portant ce nom et lisez le fichier portant ce nom. Si le document n’est pas basé sur un fichier, créez le document en fonction de l’identificateur de document lui-même. Vous devez effectuer les actions précédentes uniquement si vous envisagez d’enregistrer et de restaurer des documents.

Appelé par le Framework quand il doit obtenir une fenêtre enfant (généralement une fenêtre de vue ou de séparateur) à afficher dans une miniature de l’onglet de la barre des tâches Windows 7.

```
virtual CWnd* GetTaskbarPreviewWnd();
```

### <a name="return-value"></a>Valeur de retour

Doit retourner un pointeur valide vers un `CWnd` objet dont l’aperçu doit être affiché sur un onglet de la barre des tâches Windows 7 associé à cet enfant MDI. L’implémentation par défaut retourne une fenêtre enfant de cet enfant MDI avec l’ID de contrôle AFX_IDW_PANE_FIRST (qui `CView`est généralement une classe dérivée de).

##  <a name="gettabproxywnd"></a>  CMDIChildWndEx::GetTabProxyWnd

Retourne la fenêtre de proxy onglet inscrite avec les onglets de la barre des tâches Windows 7.

```
CMDITabProxyWnd* GetTabProxyWnd();
```

### <a name="return-value"></a>Valeur de retour

Pointeur vers un `CMDITabProxyWnd` objet, qui est inscrit avec les onglets de la barre des tâches de Windows 7.

##  <a name="enabletaskbarthumbnailcliprect"></a>  CMDIChildWndEx::EnableTaskbarThumbnailClipRect

Active ou désactive la sélection automatique d’une partie de la zone client d’une fenêtre à afficher comme miniature de la fenêtre dans la barre des tâches.

```
void EnableTaskbarThumbnailClipRect(BOOL bEnable = TRUE);
```

### <a name="parameters"></a>Paramètres

*bEnable*<br/>
Spécifie s’il faut activer (TRUE) ou désactiver (FALSe) la sélection automatique d’une partie de la zone client d’une fenêtre à afficher.

##  <a name="canshowontaskbartabs"></a>  CMDIChildWndEx::CanShowOnTaskBarTabs

Indique à l’infrastructure si cet enfant MDI peut être affiché sur les onglets de la barre des tâches de Windows 7.

```
virtual BOOL CanShowOnTaskBarTabs();
```

### <a name="return-value"></a>Valeur de retour

TRUE si le contenu de l’enfant MDI peut être affiché sur les miniatures de la barre des tâches de Windows 7.

### <a name="remarks"></a>Notes

Substituez cette méthode dans une classe dérivée et retournez FALSe pour désactiver l’apparence de cet enfant MDI sur les onglets de la barre des tâches de Windows 7.
