## <a name="remarks"></a>Notes

Si vous souhaitez créer une classe de volets qui prend en charge les `CBasePane` fonctionnalités d’amarrage étendues disponibles dans MFC, vous devez le tirer de ou de [la classe CPane](../../mfc/reference/cpane-class.md).

## <a name="customization-tips"></a>Conseils de personnalisation

Les conseils de personnalisation suivants se rapportent aux classes `CBasePane Class` et à toutes les classes qui en héritent :

- Lorsque vous créez un volet, vous pouvez appliquer plusieurs nouveaux styles :

  - AFX_CBRS_FLOAT fait flotter la vitre.

  - AFX_CBRS_AUTOHIDE permet le mode auto-cacher.

  - AFX_CBRS_CLOSE permet de fermer la vitre (cachée).

  Ce sont des drapeaux que vous pouvez combiner avec une opération un peuwise-OR.

`CBasePane`met en œuvre les méthodes boolean virtuelles suivantes pour refléter ces drapeaux: [CBasePane:CanBeClosed](#canbeclosed), [CBasePane::CanAutoHide](#canautohide), [CBasePane:CanFloat](#canfloat). Vous pouvez les remplacer dans des classes dérivées pour personnaliser leur comportement.

- Vous pouvez personnaliser le comportement d’amarrage en dominant [CBasePane::CanAcceptPane](#canacceptpane). Demandez à votre volet de retourner FALSE de cette méthode pour empêcher une autre vitre de s’y amarrer.

- Si vous voulez créer un volet statique qui ne peut pas flotter et qui empêche tout autre volet d’accoster avant elle (semblable à la barre Outlookdemo dans l’exemple OutlookDemo), créez-le comme non flottant et remplaceZ [CBasePane::DoesAllowDynInsertBefore](#doesallowdyninsertbefore) de retourner FALSE. L’implémentation par défaut renvoie FALSE si le volet est créé sans le style AFX_CBRS_FLOAT.

- Créez tous les volets avec des DIU autres que -1.

- Pour déterminer la visibilité du volet, utilisez [CBasePane::IsVisible](#isvisible). Il gère correctement l’état de visibilité dans les modes tabbed et auto-cacher.

- Si vous voulez créer un volet resizable non flottant, créez-le sans le style AFX_CBRS_FLOAT et appelez [CFrameWnd::DockControlBar](../../mfc/reference/cframewnd-class.md#dockcontrolbar).

- Pour exclure une vitre d’une disposition d’amarrage ou pour retirer une barre d’outils de sa barre de dock, appelez [CBasePane::UndockPane](#undockpane). N’appelez pas cette méthode pour les vitres en mode auto-cacher ou pour les vitres qui résident dans des onglets de fenêtres tabbed.

## <a name="cbasepanesetdockingmode"></a><a name="setdockingmode"></a>CBasePane::SetDockingMode

Définit le mode d’amarrage pour la vitre.

```cpp
void SetDockingMode(AFX_DOCK_TYPE dockModeNew);
```

### <a name="parameters"></a>Paramètres

*dockModeNew*<br/>
[dans] Spécifie le nouveau mode d’amarrage pour la vitre.

### <a name="remarks"></a>Notes

Le cadre prend en charge deux modes d’amarrage : standard et immédiat.

En mode d’amarrage standard, les vitres et les mini-fenêtres à ossature sont déplacées à l’aide d’un rectangle de traînée. Dans le mode d’amarrage immédiat, les barres de commande et les fenêtres à mini-cadre sont déplacées immédiatement dans leur contexte.

Initialement, le mode d’amarrage est défini globalement par [CDockingManager:m_dockModeGlobal](../../mfc/reference/cdockingmanager-class.md#m_dockmodeglobal). Vous pouvez définir le mode d’amarrage `SetDockingMode` pour chaque volet individuellement en utilisant la méthode.
