# &#201;criture d&#39;applications MFC
[!INCLUDE[vs2017banner](../assembler/inline/includes/vs2017banner.md)]

Cet article explique les étapes initiales d'effectuer pour développer votre application.  D'abord, vous devez déterminer quel type l'application que vous entrez.  Plusieurs options qui sont décrits dans [Choix de conception d'applications](../mfc/application-design-choices.md).  Votre application est :  
  
-   Exécuter sur Internet ou un intranet ?  
  
-   Exécuter sur un client ou sur un serveur ?  
  
-   Exécuter dans un navigateur ou comme application autonome ?  
  
-   À COM ou la technologie active ?  
  
-   Données de téléchargement à WinInet ou monikers asynchrones ?  
  
 Vos décisions déterminent les classes sont appropriées pour votre application.  L'utilisation de réponses également déterminer les sélections que vous effectuez lorsque vous exécutez l'Application pour démarrer la construction votre application.  
  
 Après avoir pris vos décisions de conception initiales sur votre application Web, vous pouvez utiliser l'Application pour commencer.  Utilisez l'Application pour créer une application squelette et modifier le code comme décrit dans les éléments suivants :  
  
-   Pour un contrôle ActiveX, consultez [Contrôles ActiveX Internet](../mfc/activex-controls-on-the-internet.md).  
  
-   Pour un document actif, consultez [Documents actifs sur Internet](../mfc/active-documents-on-the-internet.md).  
  
 Les éléments suivants fournissent également l'aide de l'instruction permettent de démarrer vos tâches de programmation :  
  
-   [Choix en matière de conception d'applications](../mfc/application-design-choices.md)  
  
-   [Monikers asynchrones sur Internet](../mfc/asynchronous-monikers-on-the-internet.md)  
  
-   [Éléments fondamentaux relatifs à WinInet](../mfc/wininet-basics.md)
