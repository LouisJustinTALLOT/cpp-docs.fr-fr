# <a name="writing-mfc-applications"></a>Écriture d'applications MFC

Cet article explique les étapes initiales à suivre pour développer votre application. Tout d’abord, vous devez décider quel type d’application vous écrivez. Plusieurs choix ont été abordés dans [choix](../mfc/application-design-choices.md)de la conception d’applications. Votre application sera-elle :

- Exécution sur Internet ou sur un intranet

- Exécution sur un client ou sur un serveur

- Exécution dans un navigateur ou en tant qu’application autonome

- Utilisation de la technologie COM ou active

- Téléchargement de données à l’aide de WinInet ou de monikers asynchrones

Vos décisions déterminent les classes qui conviennent à votre application. Vos réponses vous aident également à déterminer les sélections que vous effectuez lors de l’exécution de l’Assistant application pour commencer à construire votre application.

Une fois que vous avez pris les décisions de conception initiales de votre application Internet, vous pouvez utiliser l’Assistant application pour commencer. Utilisez l’Assistant application pour créer une application squelette et modifier le code comme décrit dans les articles suivants :

- Pour un contrôle ActiveX, consultez [contrôles ActiveX sur Internet](../mfc/activex-controls-on-the-internet.md).

Les articles suivants fournissent également des instructions pour vous aider à démarrer vos tâches de programmation :

- [Choix de conception d’application](../mfc/application-design-choices.md)

- [Monikers asynchrones sur Internet](../mfc/asynchronous-monikers-on-the-internet.md)

- [Concepts de base de WinInet](../mfc/wininet-basics.md)
