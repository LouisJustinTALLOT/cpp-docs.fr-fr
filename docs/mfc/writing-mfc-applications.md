# <a name="writing-mfc-applications"></a>Écriture d'applications MFC

Cet article explique les étapes initiales de développement de votre application. Tout d’abord, vous devez décider quel type d’application que vous écrivez. Plusieurs options ont été abordés dans [choix de conception d’Application](../mfc/application-design-choices.md). Votre application sera :

- En cours d’exécution sur Internet ou un intranet

- En cours d’exécution sur un client ou sur un serveur

- En cours d’exécution dans un navigateur ou comme application autonome

- À l’aide de COM ou la technologie Active

- Téléchargement des données à l’aide de WinInet ou des monikers asynchrones

Vos décisions de déterminent les classes qui sont appropriées pour votre application. Vos réponses permettent également de déterminer les sélections que vous effectuez lorsque vous exécutez l’Assistant Application pour commencer à générer votre application.

Une fois que vous avez effectué vos décisions de conception initiale sur votre application Internet, vous pouvez utiliser l’Assistant Application pour commencer. Utilisez l’Assistant Application pour créer une application squelette et modifier le code, comme décrit dans les articles suivants :

- Pour un contrôle ActiveX, consultez [contrôles ActiveX sur Internet](../mfc/activex-controls-on-the-internet.md).

Les articles suivants fournissent également des instructions pour vous aider à démarrer vos tâches de programmation :

- [Choix en matière de conception d’applications](../mfc/application-design-choices.md)

- [Monikers asynchrones sur Internet](../mfc/asynchronous-monikers-on-the-internet.md)

- [Notions de base de WinInet](../mfc/wininet-basics.md)
