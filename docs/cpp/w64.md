# <a name="__w64"></a>__w64

Ce mot clé spécifique à Microsoft est obsolète. Dans les versions de Visual Studio antérieures à Visual Studio 2013, cela vous permet de marquer des variables, de sorte que quand vous compilez avec [/Wp64](../build/reference/wp64-detect-64-bit-portability-issues.md) , le compilateur signale tous les avertissements signalés si vous compilez avec un compilateur 64 bits.

## <a name="syntax"></a>Syntaxe

> *identificateur* de **__w64** de *type*

### <a name="parameters"></a>Paramètres

*type*<br/>
L’un des trois types susceptibles de provoquer des problèmes dans le code est porté d’un 32-bit vers un compilateur 64 bits : **int**, **long**ou un pointeur.

*identifier*<br/>
Identificateur de la variable que vous créez.

## <a name="remarks"></a>Notes

> [!IMPORTANT]
>  L’option de compilateur [/Wp64](../build/reference/wp64-detect-64-bit-portability-issues.md) et le mot clé **__w64** sont déconseillés dans Visual Studio 2010 et Visual Studio 2013 et supprimés à partir de Visual Studio 2013. Si vous utilisez l’option `/Wp64` du compilateur sur la ligne de commande, le compilateur émet un avertissement de ligne de commande D9002. Le mot clé **__w64** est ignoré en mode silencieux. Au lieu d’utiliser cette option et un mot clé pour détecter les problèmes de portabilité 64 bits C++ , utilisez un compilateur Microsoft ciblant une plateforme 64 bits. Pour plus d’informations, consultez [configurer C++ Visual pour les cibles 64 bits, x64](../build/configuring-programs-for-64-bit-visual-cpp.md).

Tout typedef qui a **__w64** sur celui-ci doit être 32 bits sur x86 et 64 bits sur x64.

Pour détecter les problèmes de portabilité en utilisant des versions C++ du compilateur Microsoft antérieures à Visual Studio 2010, le mot clé **__w64** doit être spécifié sur tout typedefs qui modifie la taille entre les plateformes 32 bits et 64 bits. Pour tout type de ce type, **__w64** doit apparaître uniquement sur la définition 32 bits du typedef.

Pour la compatibilité avec les versions précédentes, **_w64** est un synonyme de **__w64** sauf si l’option de compilateur [/za \(désactiver les extensions de langage)](../build/reference/za-ze-disable-language-extensions.md) est spécifiée.

Le mot clé **__w64** est ignoré si la compilation n’utilise pas `/Wp64`.

Pour plus d'informations sur le portage vers 64 bits, consultez les rubriques suivantes :

- [Options du compilateur MSVC](../build/reference/compiler-options.md)

- [Portage du code 32 bits vers du code 64 bits](../build/common-visual-cpp-64-bit-migration-issues.md)

- [Configurer Visual C++ pour des cibles x64 64 bits](../build/configuring-programs-for-64-bit-visual-cpp.md)

## <a name="example"></a>Exemple

```cpp
// __w64.cpp
// compile with: /W3 /Wp64
typedef int Int_32;
#ifdef _WIN64
typedef __int64 Int_Native;
#else
typedef int __w64 Int_Native;
#endif

int main() {
   Int_32 i0 = 5;
   Int_Native i1 = 10;
   i0 = i1;   // C4244 64-bit int assigned to 32-bit int

   // char __w64 c;  error, cannot use __w64 on char
}
```
