# <a name="w64"></a>__w64

**Section spécifique à Microsoft**

Ce mot clé est obsolète. Dans les versions de Visual Studio antérieures à Visual Studio 2013, cela vous permet de marquer les variables, afin que lorsque vous compilez avec [/Wp64](../build/reference/wp64-detect-64-bit-portability-issues.md) le compilateur signale les avertissements qui seraient signalés si vous compiliez avec un compilateur 64 bits.

## <a name="syntax"></a>Syntaxe

> *type* **__w64** *identifier*

### <a name="parameters"></a>Paramètres

*type*<br/>
Un des trois types qui peut entraîner des problèmes dans le code porté à partir d’une version 32 bits à un compilateur 64 bits : **int**, **long**, ou un pointeur.

*identifier*<br/>
Identificateur de la variable que vous créez.

## <a name="remarks"></a>Notes

> [!IMPORTANT]
>  Le [/Wp64](../build/reference/wp64-detect-64-bit-portability-issues.md) option du compilateur et **__w64** mot clé sont déconseillés dans Visual Studio 2010 et Visual Studio 2013 et supprimés à partir de Visual Studio 2013. Si vous utilisez la `/Wp64` option du compilateur sur la ligne de commande, le compilateur émet l’avertissement de ligne de commande D9002. Le **__w64** mot clé est ignoré en mode silencieux. Au lieu d’utiliser cette option et un mot clé pour détecter les problèmes de portabilité 64 bits, utilisez un Microsoft C++ compilateur qui cible une plateforme 64 bits. Pour plus d’informations, consultez [configurer Visual C++ pour x64 64 64 bits, cibles](../build/configuring-programs-for-64-bit-visual-cpp.md).

Tout typedef qui a **__w64** doit être 32 bits sur x86 et 64 bits sur x64.

Pour détecter les problèmes de portabilité à l’aide de versions de Microsoft C++ compilateur antérieures à Visual Studio 2010, le **__w64** mot clé doit être spécifié sur les typedefs qui changent de taille entre 32 bits et les plateformes 64 bits. Pour un tel type, **__w64** doit apparaître uniquement sur la définition de 32 bits du typedef.

Pour assurer la compatibilité avec les versions précédentes, **_w64** est un synonyme de **__w64** , sauf si option du compilateur [/Za \(désactiver les extensions de langage)](../build/reference/za-ze-disable-language-extensions.md) est spécifié.

Le **__w64** mot clé est ignorée si la compilation n’utilise pas `/Wp64`.

Pour plus d'informations sur le portage vers 64 bits, consultez les rubriques suivantes :

- [Options du compilateur MSVC](../build/reference/compiler-options.md)

- [Portage de Code 32 bits vers 64 bits](../build/common-visual-cpp-64-bit-migration-issues.md)

- [Configurer Visual C++ pour des cibles x64 64 bits](../build/configuring-programs-for-64-bit-visual-cpp.md)

## <a name="example"></a>Exemple

```cpp
// __w64.cpp
// compile with: /W3 /Wp64
typedef int Int_32;
#ifdef _WIN64
typedef __int64 Int_Native;
#else
typedef int __w64 Int_Native;
#endif

int main() {
   Int_32 i0 = 5;
   Int_Native i1 = 10;
   i0 = i1;   // C4244 64-bit int assigned to 32-bit int

   // char __w64 c;  error, cannot use __w64 on char
}
```
