# <a name="__w64"></a>__w64

Ce mot clé spécifique à Microsoft est obsolète. Dans les versions de Visual Studio plus tôt que Visual Studio 2013, cela vous permet de marquer les variables, de sorte que lorsque vous compilez avec [/Wp64](../build/reference/wp64-detect-64-bit-portability-issues.md) le compilateur signalera tous les avertissements qui seraient signalés si vous compiliez avec un compilateur 64 bits.

## <a name="syntax"></a>Syntaxe

> *type* *__w64'identifiant* **__w64**

### <a name="parameters"></a>Paramètres

*type*<br/>
L’un des trois types qui pourraient causer des problèmes dans le code étant porté d’un 32 bits à un compilateur 64 bits: **int**, **long**, ou un pointeur.

*Identificateur*<br/>
Identificateur de la variable que vous créez.

## <a name="remarks"></a>Notes

> [!IMPORTANT]
> L’option de compilation [/Wp64](../build/reference/wp64-detect-64-bit-portability-issues.md) et le mot clé **__w64** sont dépréciés dans Visual Studio 2010 et Visual Studio 2013 et supprimés à partir de Visual Studio 2013. Si vous `/Wp64` utilisez l’option compilateur sur la ligne de commande, le compilateur émet d’avertissement de ligne de commandement D9002. Le mot clé **__w64** est ignoré silencieusement. Au lieu d’utiliser cette option et ce mot clé pour détecter les problèmes de portabilité 64 bits, utilisez un compilateur Microsoft CMD qui cible une plate-forme 64 bits. Pour plus d’informations, voir [Configure Visual CMD pour les cibles 64 bits et x64](../build/configuring-programs-for-64-bit-visual-cpp.md).

Tout typedef qui a **__w64** sur elle doit être 32 bits sur x86 et 64 bits sur x64.

Pour détecter les problèmes de portabilité en utilisant des versions du compilateur Microsoft CMD plus tôt que Visual Studio 2010, le mot clé **__w64** doit être spécifié sur tous les types qui changent de taille entre 32 bits et 64 bits plates-formes. Pour un tel type, **__w64** ne doit apparaître que sur la définition 32 bits du tapdef.

Pour la compatibilité avec les versions précédentes, **_w64** est synonyme **de __w64** à moins que l’option compilateur [/Za \(Désactiver extensions de langage)](../build/reference/za-ze-disable-language-extensions.md) est spécifiée.

Le mot clé **__w64** est ignoré si `/Wp64`la compilation n’utilise pas .

Pour plus d'informations sur le portage vers 64 bits, consultez les rubriques suivantes :

- [Options du compilateur MSVC](../build/reference/compiler-options.md)

- [Portage du code 32 bits vers du code 64 bits](../build/common-visual-cpp-64-bit-migration-issues.md)

- [Configurer Visual C++ pour des cibles x64 64 bits](../build/configuring-programs-for-64-bit-visual-cpp.md)

## <a name="example"></a>Exemple

```cpp
// __w64.cpp
// compile with: /W3 /Wp64
typedef int Int_32;
#ifdef _WIN64
typedef __int64 Int_Native;
#else
typedef int __w64 Int_Native;
#endif

int main() {
   Int_32 i0 = 5;
   Int_Native i1 = 10;
   i0 = i1;   // C4244 64-bit int assigned to 32-bit int

   // char __w64 c;  error, cannot use __w64 on char
}
```
