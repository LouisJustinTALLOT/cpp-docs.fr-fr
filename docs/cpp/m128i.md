# <a name="__m128i"></a>__m128i

**Microsoft Spécifique**

Le type de données **__m128i,** pour une utilisation avec les instructions d’extensions SIMD \<2 (SSE2) en streaming, est défini en emmintrin.h>.

```cpp
// data_types__m128i.cpp
#include <emmintrin.h>
int main() {
   __m128i x;
}
```

## <a name="remarks"></a>Notes

Vous ne devez pas accéder directement aux **champs __m128i.** Vous pouvez toutefois visualiser ces types dans le débogueur. Une variable de type **__m128i** cartes aux registres XMM[0-7].

Les variables de type **__m128i** sont automatiquement alignées sur les limites de 16-byte.

> [!NOTE]
> L’utilisation de variables de type **__m128i** fera générer le `movdqa` compilateur pour générer l’instruction SSE2. Cette instruction ne cause pas de défaut sur les processeurs Pentium III, mais `movdqa` entraînera une défaillance silencieuse, avec des effets secondaires possibles causés par toutes les instructions traduit sur les processeurs Pentium III.

Le **type de** données __m128i n’est pas pris en charge sur les processeurs ARM.

**END Microsoft Spécifique**
