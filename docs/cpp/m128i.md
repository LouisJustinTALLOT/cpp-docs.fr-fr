# <a name="__m128i"></a>__m128i

**Section spécifique de Microsoft**

Le type de données **__m128i** , à utiliser avec les instructions SSE2 (streaming SIMD Extensions 2) intrinsèques, est défini dans \<emmintrin. h >.

```cpp
// data_types__m128i.cpp
#include <emmintrin.h>
int main() {
   __m128i x;
}
```

## <a name="remarks"></a>Notes

Vous ne devez pas accéder directement aux champs **__m128i** . Vous pouvez toutefois visualiser ces types dans le débogueur. Une variable de type **__m128i** est mappée aux registres XMM [0-7].

Les variables de type **_m128i** sont alignées automatiquement sur des limites de 16 octets.

> [!NOTE]
>  L’utilisation de variables de type **__m128i** force le compilateur à générer l’instruction de `movdqa` SSE2. Cette instruction ne provoque pas d’erreur sur les processeurs Pentium III, mais entraîne une défaillance silencieuse, avec des effets secondaires possibles en raison des instructions `movdqa` converties en processeurs Pentium III.

Le type de données **__m128i** n’est pas pris en charge sur les processeurs ARM.

**Fin de la section spécifique de Microsoft**
