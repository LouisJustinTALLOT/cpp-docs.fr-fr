# Redistribution de fichiers de prise en charge de base de donn&#233;es
[!INCLUDE[vs2017banner](../assembler/inline/includes/vs2017banner.md)]

Vous pouvez redistribuer des fichiers de prise en charge pour les objets d'accès aux données \(DAO\) et pour les technologies de bases de données se trouvant dans le Kit de développement Microsoft Data Access SDK.  
  
## Installation de fichiers de prise en charge DAO  
 Pour obtenir la version la plus récente de DAO, consultez l'[Article 239114 : Comment faire pour obtenir le dernier Service Pack pour le moteur de base de données Microsoft Jet 4.0](http://go.microsoft.com/fwlink/?LinkId=198014) sur le site Web de support technique Microsoft.  
  
## Installation des fichiers de prise en charge du Kit de développement Microsoft Data Access SDK  
 Utilisez Mdac\_typ.exe pour installer la prise en charge ODBC, OLE DB, ADO \(ActiveX Data Objects\) et RDS \(Remote Data Services\).  Mdac\_typ.exe est situé dans le dossier ..\\WCU\\MDAC28\\ sur le média d'installation Visual Studio.  Vous pouvez également télécharger Mdac\_typ.exe à partir du site [Microsoft Download Center](http://go.microsoft.com/fwlink/?LinkId=198015).  
  
 Pour plus d'informations, sur le site Web [MSDN](http://go.microsoft.com/fwlink/?LinkId=198016), recherchez « redistribution MDAC 2,8 SP1 ».  Si vous utilisez des projets d'installation Visual Studio pour déployer votre application, consultez [Deployment and Dependencies](http://msdn.microsoft.com/fr-fr/49e9b84d-bd6a-4388-b9ac-46ea79cf0733).
